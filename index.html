    What this does

    Burp Payloader gives you ready-made payloads for authorized web testing. Pick a vulnerability class, copy a payload, and paste it into Burp Suite Repeater or Intruder.

    Quick start

    1. Choose a class such as SQL injection, XSS, or SSRF.
    2. Filter by technique or search for a word you recognize.
    3. Click copy next to a payload.
    4. Paste it into the request parameter you are testing in Burp.
    5. Look for a changed response, error, delay, redirect, or reflected text.

    Repeater tab

    Use Repeater when you are testing one request by hand. Copy one payload at a time and watch how the server response changes.

    Intruder tab

    Use Intruder when you want a list. Pick filters, choose an encoding if needed, then copy or download the list for a Burp Intruder payload set.

    Why Generate is disabled

    Generate one variant and Add one variant ask the local assistant for a single payload value when a model is loaded. Some small browser models still output request blocks or refuse security-testing requests even when they are authorized. When that happens, the app extracts the first usable payload or creates one fresh catalog variant.

    If your browser does not support WebGPU, generation still works from the catalog. Each click rotates and mutates the single variant so you are not stuck with the same output.

    What variants are

    Variants are extra payload ideas based on the notes in the text box, such as a JSON login request, a blocked quote character, or a reflected search parameter.

    Everything runs locally. If loaded, the assistant is downloaded once through WebGPU and cached; request details stay in the browser. Use only on systems you own or have explicit permission to test.
